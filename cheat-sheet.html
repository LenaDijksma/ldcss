Cheat sheet

Every ld-* class and data-ld-* attribute in ldcss, in one scannable page. See README.md for full markup examples.

Theming

AttributeValuesNotes
data-ld-themelight · darkOn <html> or any container
data-ld-unstyled—Resets radius/shadow/border-weight/accent-fill to neutral on a container, so your own stylesheet (loaded after ldcss.css) can restyle cleanly without fighting specificity — layout, spacing, and behavior are unaffected
data-ld-notheme—Freezes subtree at light appearance
data-ld-toggle="theme"—Click to flip theme, persists via localStorage; on a first visit with nothing saved yet, falls back to the OS/browser's prefers-color-scheme

Responsive variants

ClassBreakpointNotes
ld-sm:*min-width: 640pxSame sm breakpoint .ld-grid already uses
ld-md:*min-width: 960pxSame md breakpoint .ld-grid already uses
ld-lg:*min-width: 1200pxNew in v1.1.0
ld-xl:*min-width: 1440pxNew in v1.1.0
Each of the four prefixes above covers: d-* (display), grid-cols-* (1–6), flex-row/flex-col, text-left/center/right, gap-0/2/4/6, pad-a-*/mar-a-* (all-sides, sizes 0/2/3/4/6/8), text-xs…text-3xl, and w-25/50/75/100 — e.g. ld-md:flex-row, ld-lg:pad-a-6. Not covered: per-side padding/margin, the full 8-step spacing scale, and border-radius — same pattern, add if needed.

Segmented, chips, input affix, combobox, empty state, timeline, icons

Class / attributeNotes
data-ld-segmented, data-ld-segmentContainer + option, attribute-only (no class needed). data-ld-active="true" marks the current option; optional data-ld-target="#input" on the container syncs the value into a hidden input
data-ld-chipInteractive filter chip (contrast with the display-only ld-tag). Wrap in a container with data-ld-chips="single" for radio-style selection; omit the wrapper for independent multi-select toggles
ld-input-group, ld-input-affixIcon/unit alongside an ld-input; add data-ld-affix="right" on the group to move it to the trailing edge
data-ld-comboboxWraps an input + data-ld-combobox-list of data-ld-combobox-option items. Typing filters, ↑/↓ highlights, Enter/click selects, Escape closes. Add a data-ld-combobox-empty element for the no-matches state
ld-empty, ld-empty-icon/-title/-body/-actionsThe "no results" pattern — pure layout/type, no JS
ld-timeline, ld-timeline-itemdata-ld-variant: accent · danger. Vertical, open-ended (contrast with the fixed-stage ld-stepper); subparts ld-timeline-time/-title/-body
ld-icon, ld-icon-sm, ld-icon-lgSizing convention for inline SVGs (1em by default, tracks surrounding font-size) — not an icon set; bring your own SVG with stroke="currentColor"
data-ld-stepper, data-ld-clickable, data-ld-step-panelsMakes an existing ld-stepper interactive — see the Table/list/pagination/stepper row above for the static attributes. Pair with data-ld-toggle="step-next"/"step-prev" buttons

Grid, carousel, navbar collapse, floating labels, validation, scrollspy, events

Class / attributeNotes
ld-grid-row, ld-col-{1-12}, ld-col-auto, ld-offset-{0-11}Flexbox 12-column grid for asymmetric layouts — distinct from the even-split ld-grid. Responsive with -sm/-md/-lg/-xl infixes, e.g. ld-col-md-6, ld-offset-lg-2. Gutter via --ld-gutter (defaults to --ld-space-4)
.ld-containerNow steps its max-width per breakpoint (100% fluid → 600 → 900 → 1100 → 1320px) instead of one fixed cap — see v2.0.0 changelog for the breaking-change note
data-ld-carousel, data-ld-carousel-track/-slide/-dots/-dotPrev/next: data-ld-toggle="carousel-prev"/"carousel-next". Autoplay: data-ld-autoplay="true" + data-ld-interval (ms) — pauses on hover
data-ld-navbar-collapse, data-ld-toggle="navbar-collapse"Inline collapse (hamburger below a breakpoint, always-visible above it) — distinct from the slide-in ld-offcanvas. data-ld-collapse-at: sm · md (default) · lg · xl
ld-form-floatingLabel rises out of the field on focus/value. Input needs placeholder=" " (a real space) and must come before the <label> in markup
.ld-validateWrap a form to get automatic red/green on .ld-input/.ld-textarea via native :user-invalid/:user-valid — only after interaction, not on first paint. Independent of the manual data-ld-error/-success attributes
data-ld-shape="button"/"avatar", ld-skeleton-w-{25,50,75,100}, ld-skeleton-paragraphExtends the existing ld-skeleton/data-ld-shape="circle"/"text"
data-ld-scrollspyOn a container of <a href="#section"> links — the in-view section's link gets data-ld-active="true" (same attribute .ld-nav-link already styles). This page's own nav uses it
Roving tabindexTabs and the segmented control: only the active item is a Tab stop; ←/→/Home/End move between options within the group, activating as you go
ld:{component}:show / ld:{component}:hide — bubbling CustomEvents on modal, offcanvas, dropdown, popover, accordion, tab (show only), command, carousel (show only, fires per-slide), navbar. e.detail.trigger is the clicking element where applicable. Listen once, high up the DOM: document.addEventListener('ld:modal:show', fn)
